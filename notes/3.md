The repo has two related “input formats”: the **dataloader sample format** and the actual **`FlowMatchingModel.forward()` format**.

**Canonical Model Inputs**

Evidence: `FlowMatchingModel.forward()` defines the model contract here: [FlowMatchingModel.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingModel.py:322).

| Name | Shape | Meaning |
|---|---:|---|
| `x_rgb` | `(B, 3, H, W)` | Visual input image tensor, float in `[0, 1]`. Default config uses `H,W = 240,320` and image `rgb_WoArm_WArmObjKpts.png`. |
| `x_ict` | `(B, max_ict, ict_dim)` | Interaction-Centric Tokens: hand/object state tokens. Default `max_ict = 8`. |
| `ict_mask` | `(B, max_ict)` bool | Valid-token mask. `True` means real token; `False` means padding. |
| `x_t` | `(B, pred_horizon, action_dim)` | Flow-matching noisy action trajectory at diffusion/flow time `t`. |
| `t` | `(B, 1)` | Flow time scalar, sampled in `[0, 1]` during training. |
| `x_pcd` | optional `(B, max_ict, 64, 3)` | Optional per-token point cloud geometry. Used only when `use_pcd_features=True`. |
| `anchor_uv` | optional `(B, 2)` | Normalized image-space anchor-object center, used when region attention is enabled. |

The default `serve_bread` config sets `pred_horizon: 50`, `img_name: "rgb_WoArm_WArmObjKpts.png"`, `image_size: [240, 320]`, `single_hand: True`, and `max_ict: 8` in [HumanEgo.yaml](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/cfg/training/serve_bread/HumanEgo.yaml:45). So for that task, the common input shapes are roughly:

```text
x_rgb:     (B, 3, 240, 320)
x_ict:     (B, 8, 20)
ict_mask:  (B, 8)
x_t:       (B, 50, 10)
t:         (B, 1)
anchor_uv: (B, 2)
```

**ICT Token Format**

Evidence: both dataloader and model say `ict_dim = 20` for single-hand and `29` for dual-hand: [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:211), [FlowMatchingModel.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingModel.py:212).

Each ICT token is:

```text
[type_id, pose_in_ref, hand_relation(s), flag]
```

For single-hand:

```text
1 + 9 + 9 + 1 = 20 dims
```

For dual-hand:

```text
1 + 9 + 18 + 1 = 29 dims
```

Where:

```text
type_id:       0 pad, 1 left hand, 2 right hand, 3 anchor object, 4 other object
pose_in_ref:   normalized position xyz + 6D rotation = 9 dims
hand_relation: hand pose relative to this entity, 9 dims per hand
flag:          grasp for hand tokens, -1 for object tokens
```

Evidence for token construction is in `_build_ict()` at [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:311). Hand tokens are built at [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:336); object tokens at [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:362).

**Training Targets**

The dataloader also returns targets, though these are not direct model inputs except through flow matching:

| Name | Shape | Meaning |
|---|---:|---|
| `y_action` | `(B, K, 10)` single-hand or `(B, K, 20)` dual-hand | Future hand/EE trajectory: position, 6D rotation, grasp. |
| `y_obj_action` | `(B, K, 9)` | Optional object dynamics target. |
| `y_done` | `(B, K, 1)` | Task completion label. |
| `y_2d_trace` | `(B, K, targets, 2)` | Optional visual-foresight trace. |
| `x_ict_future` | `(B, max_ict, ict_dim)` | Optional temporal-contrastive future ICT target. |

Evidence: target construction and flattening happen in [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:446), especially `y_action` at [FlowMatchingDataloader.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingDataloader.py:582).

During training, `FlowMatchingTrainer` turns `y_action` into the flow target `x_1`, samples noise `x_0`, interpolates to `x_t`, then calls the model with `x_rgb`, `x_ict`, `ict_mask`, `x_t`, `t`, optional `x_pcd`, and optional `anchor_uv` ([FlowMatchingTrainer.py](/Users/lewis/SchoolWorks/06_Junior_Spring/research1/HumanEgo/training/FlowMatchingTrainer.py:421)).
