# Data & Preprocessing

This guide shows how to (1) download a sample Aria recording, (2) run the
HumanEgo preprocessing pipeline on it, and (3) optionally download the
precomputed results so you can skip the GPU pipeline.

The data lives on the public HuggingFace dataset
**[`Leo-TX/HumanEgo`](https://huggingface.co/datasets/Leo-TX/HumanEgo)** — no token
or login required.

---

## What's in a recording

Each recording (e.g. `serve_bread/aria/mps_serve_bread_000_vrs`) has two parts:

| Part | Contents | Size | Needed to run preprocessing? |
| --- | --- | --- | --- |
| **Input** | `sample.vrs` (raw Aria recording), `slam/` (MPS SLAM), `hand_tracking/` (MPS hand tracking), `else/` (VRS metadata) | ~0.6 GB | ✅ yes |
| **Output** | `preprocess/` — phase/SLAM/hand analyses, object masks, CoTracker + triangulation results, point clouds, visualization clips, and `all_data.tar` (the ~14k per-frame files, packed) | ~1.5 GB | ❌ generated by the pipeline |

The pipeline reads the **input** and writes everything to `preprocess/`. So you can
either download just the input and reproduce the output yourself, or download the
precomputed output too.

> **Note.** The hamer / wilor / mediapipe hand-tracking outputs are ablation-only and
> are **not** part of the release.

---

## 0. Install

```bash
pip install huggingface_hub        # for downloading
# and follow the repo README to install the full preprocessing environment
```

## 1. Download the sample recording

A small helper covers the common cases (`scripts/download_data.py`):

```bash
# (a) INPUT ONLY (~0.6 GB) — to run preprocessing yourself
python scripts/download_data.py --mode test

# (b) INPUT + PRECOMPUTED OUTPUT (~2 GB) — skip the GPU pipeline.
#     all_data.tar is auto-extracted for you.
python scripts/download_data.py --mode full
```

Both download into `./data/` by default. Pass `--out <dir>` to choose another location
(e.g. `python scripts/download_data.py --mode test --out /path/to/data`).

<details>
<summary>Prefer plain <code>huggingface_hub</code> (no helper script)?</summary>

```python
from huggingface_hub import snapshot_download

# input only
snapshot_download(
    repo_id="Leo-TX/HumanEgo", repo_type="dataset", local_dir="./data",
    allow_patterns=["serve_bread/aria/mps_serve_bread_000_vrs/*"],
    ignore_patterns=["serve_bread/aria/mps_serve_bread_000_vrs/preprocess/*"],
)
```

Drop `ignore_patterns` to also fetch the precomputed `preprocess/` output, then unpack
the per-frame archive:

```bash
cd data/serve_bread/aria/mps_serve_bread_000_vrs/preprocess
tar -xf all_data.tar      # recreates all_data/   (delete the tar afterwards if you like)
```
</details>

## 2. Run preprocessing

```bash
python -m preprocess.Preprocess \
    --mps_path ./data/serve_bread/aria/mps_serve_bread_000_vrs \
    --task serve_bread
```

Results are written to `…/mps_serve_bread_000_vrs/preprocess/`. If you downloaded the
precomputed output in step 1(b), you can compare against it.

---

## Downloading more data

```bash
# everything, but skip the large per-frame all_data.tar archives
python scripts/download_data.py --mode all

# everything, including all_data.tar (very large), auto-extracted
python scripts/download_data.py --mode all --with-tar
```

`all_data.tar` is just the per-frame files packed into one archive (so the dataset
stays at a few dozen files per recording instead of hundreds of thousands). It is fully
regenerable by running preprocessing, so you only need it if you want the precomputed
per-frame results without running the pipeline.
